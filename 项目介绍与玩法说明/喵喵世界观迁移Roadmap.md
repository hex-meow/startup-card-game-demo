# 喵喵世界观迁移 Roadmap

## 目标

在不改变游戏玩法的基础上，把现实创业直述世界观迁移到虚构的喵喵世界观中。玩家看到的是猫咪工坊、喵喵币、喵史事件和虚构产品；现实历史事件作为内部映射依据，用于保证故事节奏和教育价值。

## 迁移边界

| 范围 | 本轮要求 | 说明 |
| --- | --- | --- |
| 玩法规则 | 不变 | 行动力、资金、血量、回合、抽牌、商店、奖励、事件数值不改。 |
| 内容文案 | 迁移 | 路线、关卡、卡牌、事件、说明文字改为喵喵世界表达。 |
| 内容生产 | 新增工具 | 提供可复用的现实内容到喵喵世界观的转换工具。 |
| 美术资产 | 暂不重制 | 当前 PNG 文件可继续占位，后续统一生成喵化资产。 |
| 导出物 | 暂不重制 | PPT/DOCX/截图输出等内容稳定后再生成。 |

## 内容转换工作流

1. 收集现实创业素材或旧版内容。
2. 用 `tools/convert_to_meow_world.py` 做第一轮名称和语气转换。
3. 人工检查历史映射是否准确、喵化语气是否统一、是否保留教学价值。
4. 把通过审核的内容填入内容编辑器或 `app.js` 默认数据。
5. 运行静态页面，检查路线、关卡、事件、商店、奖励和最终报告是否一致。

## 验收清单

- 玩家可见主标题和路线名称不再以现实公司或现实产品为主称呼。
- 玩法数值没有因为内容迁移发生非预期变化。
- 每个现实历史节点都有对应的喵史事件名和玩家可见解释。
- 同一现实实体在所有卡牌、事件和文档中使用同一个喵化名称。
- 文档、Demo、转换工具和 roadmap 使用同一套术语。
